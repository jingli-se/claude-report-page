对话要点 / Conversation Key Points (from pasted thread)

背景确认 · Background confirmed  —  Temu（Yifan）

Temu（Yifan）确认：CPA模型目前少学习约25%的激活样本。原因是A2W2A流量没有配置MMP点击链路，Adjust无法对这部分流量归因，导致 actv 信号无法进入模型训练。Temu内部真实转化率与模型转化率的差距约为25%。

Temu (Yifan) confirmed: the CPA model is under-learning ~25% of activation signals because A2W2A traffic has no MMP click link configured → Adjust cannot attribute it → actv is excluded from training.

Install signal coverage — 三类信号分布
actv only — Temu S2S激活，无Adjust归因。Liftoff已能收到，但未被识别为install进入ML。
MMP + actv overlap — 同时有Adjust归因和Temu S2S激活的用户，去重逻辑已覆盖。
MMP install only — 只有Adjust归因、没有S2S actv的用户。
Jing的猜测 · Hypothesis — 待确认

25%的缺口 = MMP install only 这部分用户。具体来说：A2W2A campaign 在 Temu 设想的多点击模板完成之后，有一部分用户被 Adjust 链接直接带到商店完成安装。这部分安装有 MMP 归因，但没有对应的 S2S actv，因此只出现在 MMP install only 区域，Liftoff 的 CPA 模型当前未能学到这部分信号。

Hypothesis to verify with Temu: is the ~25% gap specifically the "MMP install only" segment — users who installed via Adjust link directly, with no corresponding S2S actv postback?

去重前提疑问 · Dedup prerequisite challenged  —  Temu（cris.chen）↔ Liftoff（Yifan）

Temu（cris.chen）提出质疑：S2S的 actv 改名为 install 后仍走S2S链路，不经过Adjust——去重的依据是什么？

Liftoff（Yifan）回应：去重必须依赖Adjust的install信号，A2W2A侧必须先配置MMP点击链路才能归因，去重才能生效。但Temu（cris.chen）表示MMP点击链路配置"还在评估中"，造成进展阻塞。

Liftoff（Yifan）："de-duplication 必须依赖adjust的install信号，所以没太理解这里为什么还有评估的事情？"
方案扩展讨论 · Scope expansion raised  —  Liftoff（Jing & Yifan）转述 Robert 意见

Liftoff（Jing & Yifan）转述 Robert 的问题：希望在MMP计划中也做测试，将部分 actv 改名为 install 回传给Liftoff，一方面验证去重是否生效，另一方面解决MMP侧的25%缺口。

Liftoff（Jing & Yifan）进一步解释：推动actv+install并集方案的初始动因，正是希望弥补CPA模型少学习的这25%激活样本。

Temu（cris.chen）转述 Liftoff："今天早上Robert老师也想问你们mmp的一部分campaign要不要给一些actv改成install的信号我们看看，这样MMP的campaign那些也不耽误，同时看dedup的是不是work"

Liftoff (Jing & Yifan) relayed Robert's ask: test dedup on MMP campaigns by renaming some actv → install, to validate dedup works and close the 25% training gap.

最新更新 · Latest update  —  Temu（cris.chen）明确提出两个方向

Temu（cris.chen）针对MMP计划的actv+install并集，明确提出两个处理方向，并强调不希望用"改名"方式破坏现有信号体系：

Temu（cris.chen）："1. 直接你方将mmp里的actv信号当中install信号使用做去重，这样我方就不用做任何调整就可完成测试；2. 如果你们不想改，我们不希望使用改名的方式做这件事，而是多回传一路install信号给你们，这样不会破坏现状"

Temu (cris.chen) proposed two options: (1) Liftoff treats MMP actv as install internally — zero Temu changes needed; (2) If Liftoff prefers not to change, Temu will send an additional separate install postback — preserving the existing signal without renaming.
